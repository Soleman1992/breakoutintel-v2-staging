# BreakoutIntel — NSE/BSE Swing Trading Platform

## Root Cause Analysis: Why The Previous Dashboard Had Wrong Data

### Problem
The previous dashboard was **100% simulated** — all numbers were generated by `Math.random()`. 
No actual API calls to NSE, Yahoo Finance, or any market data provider were made.

```javascript
// OLD (BROKEN) — pure fiction
var STOCKS = [{ sym:'DIXON TECH', base:14280 }]  // hardcoded, never updates
function ji(base, pct){ return base*(1+rnd(-pct,pct)) } // fake jitter
```

### Root Causes
1. Browser CORS policy blocks direct calls to NSE/Yahoo from frontend
2. No backend server existed to proxy real data
3. No WebSocket server for streaming
4. No database to persist scans/portfolios
5. All "live" data was Math.random() noise around hardcoded base prices

### Solution Architecture
```
[NSE Website + Yahoo Finance] ──► [Node.js Backend]
         (Free, no key)                    │
                                    [Redis Cache]
                                           │
                                   [WebSocket Server] ──► [Next.js Frontend]
                                           │
                                   [PostgreSQL DB]
```

## Deployment Options (All Free)

### Option 1: Render.com (Easiest, 100% Free)
```bash
# 1. Push to GitHub
git init && git add . && git commit -m "BreakoutIntel v1"
git remote add origin https://github.com/yourusername/breakoutintel.git
git push

# 2. Go to render.com → New → Blueprint → Connect repo
# Render auto-reads render.yaml and creates all services

# 3. Set env vars in Render dashboard:
# - ANTHROPIC_API_KEY
# - REDIS_URL (get free from upstash.com)
```

### Option 2: VPS Self-Hosted (Hetzner €4/mo, Oracle FREE)
```bash
# Oracle Cloud Free Tier: 4 ARM CPUs, 24GB RAM, 200GB storage — FOREVER FREE
# Get at: cloud.oracle.com → Always Free tier

# SSH into your VPS
git clone https://github.com/yourusername/breakoutintel.git
cd breakoutintel
cp .env.example .env
nano .env  # Fill in values

chmod +x scripts/deploy.sh
./scripts/deploy.sh
```

### Option 3: Vercel (Frontend) + Railway (Backend) 
```bash
# Frontend → Vercel (free)
cd frontend && vercel deploy

# Backend → Railway ($5 trial, then $5/mo)
cd backend && railway up

# Database → Supabase (free 500MB PostgreSQL)
# Cache → Upstash Redis (free 10k req/day)
```

## Future Module Additions (Plug-in Architecture)

Each new feature is a new service file in `backend/src/services/`:

```javascript
// Adding F&O Scanner module:
// 1. Create backend/src/services/options.js
// 2. Add route in backend/src/routes/options.js
// 3. Mount route in backend/src/index.js
// 4. Create frontend component src/app/components/OptionsChain.tsx
// No need to rebuild anything else.
```

Planned modules (plug in without disrupting existing system):
- `services/options.js` — F&O scanner, option chain, PCR
- `services/fii.js` — FII/DII institutional flow tracking
- `services/insider.js` — SEBI insider trading alerts
- `services/backtest.js` — Strategy backtesting engine
- `services/alerts.js` — Push notifications (email/Telegram)
- `services/mobile.js` — API for React Native mobile app
- `services/saas.js` — Multi-tenant, subscription management
